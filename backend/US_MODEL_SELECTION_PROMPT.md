# 美国主流大模型选型咨询 Prompt

---

## 背景信息

我正在开发一个**中文内容管理系统（CMS）的校对服务**，需要选择最适合的大语言模型来处理以下任务：

### 核心功能需求

1. **中文文本校对**
   - 检测错别字、标点符号错误
   - 识别用词不当、易混淆词语
   - 检查非正式用语、网络流行语
   - 验证译名规范、特殊规范

2. **语义理解与分析**
   - 理解上下文语境
   - 判断用词是否恰当
   - 分析语气和风格一致性
   - 检测逻辑连贯性

3. **结构化输出**
   - 返回 JSON 格式的检测结果
   - 包含：规则ID、问题描述、修正建议、严重程度、置信度等字段
   - 需要稳定、可靠的输出格式

### 技术架构

- **双重检测系统**：AI 检测 + 384条确定性脚本规则
- **语言**：主要处理**繁体中文**和简体中文
- **文章长度**：通常 500-2000 字
- **处理量**：预计每月数万篇文章
- **API 调用**：异步调用，需要较高的稳定性和可靠性

### 当前使用模型

- **Claude 3.5 Sonnet (2024-10-22)**
- 优势：稳定、结构化输出好、中文理解良好
- 考虑升级原因：希望找到在**中文任务上表现更优**的模型

## 评估标准

请按以下维度评估和对比这些模型：

### 1. 中文能力（权重：40%）

- **繁体中文理解**：能否准确理解繁体中文语境
- **简体中文支持**：简繁转换的准确性
- **成语、俗语识别**：对中文特有表达的理解
- **语义细微差别**：能否区分近义词的微妙差异
- **上下文分析**：多句子、段落级别的语境理解

### 2. 文本校对能力（权重：30%）

- **语法错误检测**：语法、用词、搭配错误
- **风格一致性**：语气、正式度判断
- **逻辑连贯性**：句子间逻辑关系
- **修正建议质量**：给出的建议是否准确、实用

### 3. API 性能（权重：20%）

- **响应速度**：平均延迟（ms）
- **稳定性**：API 可用性、错误率
- **并发能力**：支持的最大并发请求数
- **Token 限制**：上下文窗口大小

### 4. 成本效益（权重：10%）

- **价格**：每百万 tokens 的成本
- **免费额度**：是否有免费试用
- **计费方式**：输入/输出分开计费的合理性

### 5. 技术特性

- **结构化输出**：JSON mode 支持、格式稳定性
- **Function Calling**：函数调用能力
- **Temperature 控制**：输出稳定性可控性
- **系统提示词支持**：对复杂 system prompt 的处理
