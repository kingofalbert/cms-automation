## Error Inventory (by priority)

### P0 - Critical

### P1 - High

**1. QueuePool exhaustion causing cascading 500s** (104 app errors + 65 HTTP 5xx = 169 total)
- **Events**: `unexpected_error` (59), `google_drive_sync_item_failed` (38), `worklist_sync_background_failed` (7)
- **Endpoints affected**: `/v1/pipeline/auto-publish/{uuid}/status` (36+39), `/v1/worklist/sync` (23+26)
- **Root cause**: Long-lived DB sessions during AI pipeline processing (2-5 min/doc) within the sync loop exhaust the QueuePool (pool_size=5, max_overflow=5 = 10 max). Status polls and subsequent sync upserts wait 30s at pool_timeout then crash.
- **Cross-cutting**: All recurring errors trace to this single root cause.
- **Fix**: 4 complementary fixes applied in worktree (see Auto-Fix Results). The sync-in-progress guard + asyncio.create_task are the highest-impact changes.

### P2 - Medium

**2. asyncpg TimeoutError on sync upserts** (35 errors)
- **Source**: `sync_service.py` → `_upsert_worklist_item` → `session.flush()`
- **Root cause**: DB-side query execution exceeds `command_timeout=60s`. Distinct from pool exhaustion — connection was obtained but the query timed out (likely Supabase load or row-level lock contention).
- **Fix**: TimeoutError added to transient retry set (Fix 1). Empty error field fixed (Fix 2).

**3. Pipeline tasks stuck in "processing" indefinitely** (2 tasks, 100+ polls)
- **Tasks**: `5f585d9d` (51 polls, 4h10m), `36c52b48` (49 polls, 4h05m)
- **Root cause**: `asyncio.create_task()` fire-and-forget with no timeout/heartbeat. If instance recycles or task hangs silently, PipelineTask stays in `"processing"` forever. GAS polls every 5 min with no max retry.
- **Fix**: Tier3-ReportOnly. Needs stale task reaper (architectural decision required).

**4. BackgroundTasks + BaseHTTPMiddleware blocking sync HTTP response** (37 slow requests, avg 21s)
- **Root cause**: Starlette's `BaseHTTPMiddleware` prevents `BackgroundTasks` from truly running in background — holds HTTP response open until task completes.
- **Fix**: Fix 3 changes to `asyncio.create_task()`, decoupling sync from request lifecycle.

**5. Deployment env vars override pool size settings** (config inconsistency)
- **Root cause**: 3/12 auto-fix reduced pool defaults in `settings.py` but `deploy-backend.sh` still set `POOL_SIZE=5,OVERFLOW=5`.
- **Fix**: Fix 6 updates deploy script to `POOL_SIZE=3,OVERFLOW=2`.
